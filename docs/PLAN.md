# GitHub 偵察機 - 實作計畫

## 核心流程：從打字到看到照片

這是一個像「點餐」一樣的過程，我們把它拆成四個小關卡：

### 1. 準備好盒子 (準備輸入框與按鈕) - ✅ 已完成
- **metaphor (比喻):** 就像在餐廳桌上放一個「點餐單盒子」和一個「服務鈴」。
- **目標:** 讓使用者可以寫下想找的人的名字。

### 2. 寫下名字 (資料雙向綁定) - ✅ 已完成
- **metaphor (比喻):** 當你在點餐單上寫字，「盒子」裡的內容就會跟著變。
- **目標:** 程式會隨時記住你現在輸入的內容。

### 3. 按下按鈕，出發找人 (觸發事件與 API 請求) - ✅ 已完成
- **metaphor (比喻):** 當你按下「服務鈴」，服務生（程式）就會拿著你的點餐單，跑到大廚（GitHub 伺服器）那邊問：「嘿！有沒有這個人的照片？」
- **目標:** 點擊按鈕後，程式會去網路上抓資料。

### 4. 拿到照片，變出來！ (顯示結果) - ✅ 已完成
- **metaphor (比喻):** 服務生跑回來了！如果大廚有給照片，我們就把照片從「外帶盒」拿出來，貼在牆上。
- **目標:** 把抓到的網址放到圖片標籤裡，讓照片顯示出來。
